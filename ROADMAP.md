# PRD Agent Team Roadmap

This roadmap documents planned and completed features for this project.

The format is based on and generated by [Structured Roadmap](https://github.com/grokify/structured-roadmap), which provides a machine-readable JSON intermediate representation with deterministic Markdown generation.

## Overview

PRD Agent Team is a multi-agent system for creating, reviewing, scoring, and iteratively improving Product Requirements Documents (PRDs). It uses 16 specialized agents coordinated via DAG workflow to produce high-quality PRDs with automated revision loops.

## Architecture

The system consists of multiple layers:

- **structured-requirements**: Core PRD/MRD/TRD types, scoring, views, persona library (grokify/structured-requirements)
- **Multi-Agent Spec**: Platform-agnostic agent and team definitions in JSON/Markdown
- **agentkit**: Code generator that produces standalone Cobra CLIs from specs
- **prdtool**: CLI for PRD management (init, show, validate, score, add)
- **prdtool-mcp**: MCP server for AI assistant integration

This architecture allows the CLI to be used by any AI assistant (Kiro, Claude Code, Cursor) that can execute shell commands or connect via MCP.

## PRD Creation Workflow

The planned workflow for PRD creation:

- **1. Intake**: Import existing docs or start interactive Q&A
- **2. Score**: Run 10-category scoring to identify gaps
- **3. Question**: Agent-led questions for weak categories
- **4. Update**: Incorporate answers into PRD
- **5. Iterate**: Repeat until score >= 8.0 threshold
- **6. Output**: Generate PRD.json, PM view, Exec view

## Agent Overview

The 16 agents are organized into phases:

- **Discovery Phase**: problem-discovery, user-research, market-intel
- **Solution Phase**: solution-ideation
- **Specification Phase**: requirements, ux-journey, tech-feasibility, metrics-success
- **Risk Phase**: risk-compliance
- **Synthesis Phase**: prd-lead
- **Review Phase**: review-board, prd-scoring, exec-explainability
- **Revision Phase**: issue-extractor, revision-planner, change-validator
